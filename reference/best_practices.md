# REFERENCE: 설계 베스트프랙티스 (정적)

## 요구사항
- INVEST(독립·협상가능·가치·추정가능·작게·테스트가능)
- NFR을 FR과 분리하고 측정가능하게(예: "빠름" X → "p95 < 300ms" O)

## 아키텍처
- 단일책임·낮은결합·높은응집
- 인터페이스 계약 우선(구현 전 API/이벤트 계약 고정)
- C4: Context → Container → Component → (Code)

## 데이터
- 정규화 후 필요시 선택적 비정규화
- 조회 패턴 기반 인덱스, 무분별한 인덱스 금지
- PII 최소수집·암호화·보존기간 명시

## 테스트
- 피라미드: unit 다수 · integration 중간 · e2e 소수
- 요구별 추적, NFR별 대응(성능→부하, 보안→침투)

## 결정 기록
- ADR: 맥락 → 결정 → 대안 → 결과. 기각 사유를 남긴다.
