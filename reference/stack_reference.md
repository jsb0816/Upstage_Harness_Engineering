# REFERENCE: 스택 비교 (정적 번들)

실시간 검색을 대체하는 정적 근거. advocate/challenger/judge가 인용한다.
※ 스냅샷이므로 최신성은 로드맵(실시간 RAG)에서 보강.

## Backend
| 후보 | 강점 | 약점 | 적합 |
|---|---|---|---|
| FastAPI (Python) | 빠른 개발·타입·자동문서 | CPU 바운드 약함 | AI/데이터 연계, 중소규모 |
| Node/NestJS | JS 생태계·실시간 | 콜백·타입 관리 | 실시간·풀스택 JS |
| Spring Boot | 성숙·엔터프라이즈 | 무겁·러닝커브 | 대규모·규제 산업 |
| Go | 성능·동시성·배포 간단 | 생태계 상대적 작음 | 고성능 API·인프라 |

## Frontend
| 후보 | 강점 | 약점 |
|---|---|---|
| React | 생태계·인력 풀 | 보일러플레이트 |
| Vue | 완만한 러닝커브 | 대형 생태계 상대 열세 |
| Svelte | 번들 작음·빠름 | 생태계 작음 |

## DB
| 후보 | 강점 | 약점 | 적합 |
|---|---|---|---|
| PostgreSQL | 범용·JSON·확장 | 수평확장 수동 | 대부분의 트랜잭션 |
| MySQL | 익숙·안정 | 고급기능 상대 열세 | 표준 웹 |
| MongoDB | 스키마 유연 | 트랜잭션·조인 약점 | 비정형·초기 |
| Redis | 초고속 캐시/큐 | 영속 주의 | 캐시·세션·큐 |
